First of all, I did not come up with this title, but once I saw it on the eventbrite page, I thought I'd run with it. CSS seems to be a rather polarising technology, some people hate it, some people love it, many people are frustrated with it from time to time. But it's really not magic. Though it does take some time wrap your head around it.
The focus of today's talk will be on layout-related properties, specifically a set of properties I like to talk about as a unit, namely Flexbox, Grid and Box Alignment. Because as useful as CSS properties are on their own, the nature of CSS means that they work much better when used together. CSS is a team sport. And if there's enough time, we'll probably cover Shapes as well.

Demo time

I'm going to switch over to a stripped down set of examples to showcase these properties and also make it easier to demonstrate what DevTools can do.

Flex shorthand

flex: initial flex: 0 1 auto, cannot grow but can shrink when there isn't enough space
flex: auto flex: 1 1 auto, can grow and shrink to fit available space
flex: none flex: 0 0 auto, cannot grow or shrink, AKA inflexible
flex: <positive-number> flex: <positive-number> 1 0, can grow and shrink, extent of growth depends on flex factor
